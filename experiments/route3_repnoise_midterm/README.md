# Route 3 RepNoise Midterm Artifacts

## Context
- **Project:** CS639 harmful fine-tuning defense evaluation
- **Experiment:** Route 3 / Harmful fine-tuning attack on a trained RepNoise model
- **Purpose:** Midterm presentation evidence bundle

## What is in this folder
- `CS639_Route3V2.ipynb`: Colab notebook used to run the route.
- `artifacts/download_bundle/`: copied outputs/checkpoints/data bundle from the run.
- `artifacts/colab_stage3_bundle/`: copied stage-3 code bundle used in Colab.
- `NOTEBOOK_OVERVIEW.md`: stage-by-stage notebook walkthrough.
- `RESULTS_SUMMARY.md`: evidence-based result summary and limitations.
- `ARTIFACT_MANIFEST.md`: path-level map of key artifacts.

## How to read this folder
1. Start with `RESULTS_SUMMARY.md` for key numbers and claims.
2. Read `NOTEBOOK_OVERVIEW.md` to understand execution flow.
3. Use `ARTIFACT_MANIFEST.md` to locate raw files that back each claim.
4. Open `CS639_Route3V2.ipynb` for exact commands and environment setup.

## What this does NOT prove
- It does **not** prove full hyperparameter robustness (not a broad sweep).
- It does **not** prove cross-method superiority over all defenses.
- It is **midterm evidence**, not final comprehensive evaluation.
- It does **not** independently prove utility preservation quality beyond the artifacts provided.
