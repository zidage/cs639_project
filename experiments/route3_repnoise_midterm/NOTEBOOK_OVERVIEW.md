# Notebook Overview (`CS639_Route3V2.ipynb`)

## What the notebook appears to run
- Sets up a Colab GPU runtime and mounts Google Drive.
- Clones `zidage/cs639_project` and checks out `yurun_env_setup_sft_baseline`.
- Reinstalls specific package versions for compatibility.
- Prepares Antidote repo paths and copies prior RepNoise assets from Drive.
- Runs a harmful fine-tuning attack step on a RepNoise-trained LoRA checkpoint.
- Runs safety generation + sentiment evaluation on poisoned prompts.
- Prints baseline vs attacked safety score summary from sentiment eval JSONs.
- Bundles outputs (`ckpt/`, `data/`) into a downloadable artifact folder.

## Main stages in order
1. **Environment checks** (`torch.cuda.is_available`, GPU print).
2. **Drive mount** for artifact source and destination.
3. **Repo checkout** (`git clone`, branch checkout).
4. **Dependency pinning** (PEFT/Transformers/Datasets stack).
5. **HF token setup** (token file creation in Antidote repo).
6. **Artifact bootstrap** from Drive into `/content/cs639_project/Antidote`.
7. **Dataset smoke check** (`PKU-Alignment/BeaverTails`, first 2 rows).
8. **Attack fine-tuning run** via `train.py`.
9. **Safety generation** via `poison/evaluation/pred.py`.
10. **Safety scoring** via `poison/evaluation/eval_sentiment.py`.
11. **Score extraction** baseline vs attacked from JSON suffix entries.
12. **Bundle/export** into `download_bundle` and zip upload to Drive.

## Important datasets/configs/models mentioned
- **Base model:** `meta-llama/Llama-2-7b-hf`
- **Defense checkpoint input:** `ckpt/Llama-2-7b-hf_repnoise_0.1_0.001`
- **Attack dataset:** `PKU-Alignment/BeaverTails_dangerous`
- **Benign task/data:** `sst2` / `data/sst2.json` path conventions
- **Attack config in notebook command:** poison ratio `0.2`, sample count `1000`, LR `1e-4`, quick epoch variant (`_quick`)
- **Safety evaluation script:** `poison/evaluation/eval_sentiment.py`

## Outputs generated by the route
- Attacked adapter checkpoint under `ckpt/sst2/..._f_0.2_1000_1e-4_quick/`
- Poisoned generations and corresponding `*_sentiment_eval.json` under `data/poison/...`
- A baseline score file and attacked score file used for direct comparison
- Downloadable route bundle (`download_bundle`) and zip

## Important paths/assumptions
- Runtime assumes Colab path layout under `/content/...`.
- Notebook expects source artifacts to already exist in Drive (`/content/drive/MyDrive/CS639_RepNoise`).
- Branch hardcoded: `yurun_env_setup_sft_baseline`.
- A HuggingFace token is required for gated model/data access (token redacted in this repo copy for safety).
